## Default encoding of scalars

As in JSON, there is some [default handling](https://typespec.io/docs/libraries/http/encoding#bytes) of the common scalars like `utcDateTime`:

| Scalar Type      | Default Encoding  | Encoding name                           |
| ---------------- | ----------------- | --------------------------------------- |
| `utcDateTime`    | `xs:dateTime`     | `TypeSpec.Xml.Encoding.xmlDateTime`     |
| `offsetDateTime` | `xs:dateTime`     | `TypeSpec.Xml.Encoding.xmlDateTime`     |
| `plainDate`      | `xs:date`         | `TypeSpec.Xml.Encoding.xmlDate`         |
| `plainTime`      | `xs:time`         | `TypeSpec.Xml.Encoding.xmlTime`         |
| `duration`       | `xs:duration`     | `TypeSpec.Xml.Encoding.xmlDuration`     |
| `bytes`          | `xs:base64Binary` | `TypeSpec.Xml.Encoding.xmlBase64Binary` |

## 1. Basic properties

### 1.1 — Simple model with scalar properties

Properties of scalar types are serialized as child XML elements by default.

```tsp
model Book {
  id: integer;
  title: string;
  author: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<Book>
  <id>0</id>
  <title>string</title>
  <author>string</author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
    author:
      type: string
```

</TabItem>
</Tabs>

### 1.2 — Renaming a property for XML

`@encodedName("application/xml", ...)` or `@Xml.name(...)` changes how a property name appears in the XML output without affecting the TypeSpec model name.

<Tabs>
<TabItem label="Using @Xml.name">

```tsp
model Book {
  id: integer;

  @Xml.name("book-title")
  title: string;

  author: string;
}
```

</TabItem>
<TabItem label="Using @encodedName">

```tsp
model Book {
  id: integer;

  @encodedName("application/xml", "book-title")
  title: string;

  author: string;
}
```

</TabItem>
</Tabs>

<Tabs>
<TabItem label="XML">

```xml
<Book>
  <id>0</id>
  <book-title>string</book-title>
  <author>string</author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
      xml:
        name: "book-title"
    author:
      type: string
```

</TabItem>
</Tabs>

</TabItem>

## 2. Array of primitive types

### 2.1 — Unwrapped primitive array

Each element is serialized using the property name as its XML element name. `@Xml.unwrapped` prevents wrapping in a container element.

```tsp
model Pet {
  @Xml.unwrapped
  tags: string[];
}
```

<Tabs>
<TabItem label="XML">

```xml
<Pet>
  <tags>abc</tags>
  <tags>def</tags>
</Pet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      items:
        type: string
        xml:
          name: tags
```

</TabItem>
</Tabs>

### 2.2 — Wrapped primitive array with custom wrapper name

Without `@Xml.unwrapped`, the array is wrapped in a container element. `@encodedName` on the property customizes the wrapper element name.

```tsp
model Pet {
  @encodedName("application/xml", "ItemsTags")
  tags: string[];
}
```

<Tabs>
<TabItem label="XML">

```xml
<Pet>
  <ItemsTags>
    <string>abc</string>
    <string>def</string>
  </ItemsTags>
</Pet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      xml:
        name: "ItemsTags"
        wrapped: true
      items:
        type: string
        xml:
          name: string
```

</TabItem>
</Tabs>

### 2.3 — Unwrapped primitive array with custom item name

Using `@encodedName` on a named scalar controls the element name of each item.

```tsp
@encodedName("application/xml", "ItemsName")
scalar tag extends string;

model Pet {
  @Xml.unwrapped
  tags: tag[];
}
```

<Tabs>
<TabItem label="XML">

```xml
<Pet>
  <ItemsName>abc</ItemsName>
  <ItemsName>def</ItemsName>
</Pet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      items:
        type: string
        xml:
          name: tags
```

</TabItem>
</Tabs>

### 2.4 — Wrapped primitive array with custom wrapper and item names

Both the wrapper element and each item element have custom names via `@encodedName`.

```tsp
@encodedName("application/xml", "ItemsName")
scalar tag extends string;

model Pet {
  @encodedName("application/xml", "ItemsTags")
  tags: tag[];
}
```

<Tabs>
<TabItem label="XML">

```xml
<Pet>
  <ItemsTags>
    <ItemsName>abc</ItemsName>
    <ItemsName>def</ItemsName>
  </ItemsTags>
</Pet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      xml:
        name: "ItemsTags"
        wrapped: true
      items:
        type: string
        xml:
          name: ItemsName
```

</TabItem>
</Tabs>

## 3. Complex array types

### 3.1 — Unwrapped array of models

Each model item uses its model's XML name (`XmlTag`) when the array is unwrapped.

```tsp
model Pet {
  @Xml.unwrapped
  tags: Tag[];
}

@encodedName("application/xml", "XmlTag")
model Tag {
  name: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<Pet>
  <XmlTag>
    <name>string</name>
  </XmlTag>
</Pet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Tag:
  type: "object"
  properties:
    name:
      type: "string"
  xml:
    name: "XmlTag"
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      items:
        allOf:
          - $ref: "#/definitions/Tag"
        xml:
          name: tags
```

</TabItem>
</Tabs>

### 3.2 — Wrapped array of models (default)

Without `@Xml.unwrapped`, the array is wrapped in a container element named after the property.

```tsp
model Pet {
  tags: Tag[];
}

@encodedName("application/xml", "XmlTag")
model Tag {
  name: string;
}
```

<Tabs>
<TabItem label="XML">

```xml

      <name>string</name>
    </XmlTag>
  </tags>
</Pet>
```

</TabItem>
<TabItem label="OpenAPI3">

      type: "string"

    name: "XmlTag"

### 3.3 — Unwrapped array of models with custom item name

Using `@encodedName` on the property overrides each item's element name, taking precedence over the model's XML name.

```tsp

  @encodedName("application/xml", "ItemsTag")
  @Xml.unwrapped
  tags: Tag[];
}

@encodedName("application/xml", "XmlTag")
model Tag {
  name: string;
}
```

</TabItem>

## 4. Nested models

### 4.1 — Basic nested model

A property referencing another model is serialized as a nested XML element using the property name.

```tsp
model Book {
  author: Author;
}

model Author {
  name: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<Book>
  <author>
    <name>string</name>
  </author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    author:
      $ref: "#/components/schemas/Author"
Author:
  type: object
  properties:
    name:
      type: string
```

</TabItem>
</Tabs>

### 4.2 — Nested model with custom XML name on the model

When the referenced model has `@encodedName`, the property name still takes precedence for the XML element name.

```tsp
model Book {
  author: Author;
}

@encodedName("application/xml", "XmlAuthor")
model Author {
  name: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<Book>
  <author>
    <name>string</name>
  </author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    author:
      allOf:
        - $ref: "#/components/schemas/Author"
      xml:
        name: "author" # Here we have to redefine this name otherwise in OpenAPI semantic the `XmlAuthor` name would be used
Author:
  type: object
  properties:
    name:
      type: string
  xml:
    name: "XmlAuthor"
```

</TabItem>
</Tabs>

### 4.3 — Nested model with custom XML name on the property

Using `@encodedName` on the property overrides the element name for the nested model.

```tsp
model Book {
  @encodedName("application/xml", "xml-author")
  author: Author;
}

model Author {
  name: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<Book>
  <xml-author>
    <name>string</name>
  </xml-author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml

  type: object
  properties:
    author:
      allOf:
        - $ref: "#/components/schemas/Author"

        name: "xml-author"

  type: object
  properties:
    name:
      type: string
```

</TabItem>
</Tabs>

## 5. Attributes

### 5.1 — XML attributes and element name override

`@Xml.attribute` serializes a property as an XML attribute instead of a child element. `@Xml.name` overrides the XML element name.

```tsp
model Book {
  @Xml.attribute
  id: integer;

  @Xml.name("xml-title")
  title: string;

  author: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<Book id="0">
  <xml-title>string</xml-title>
  <author>string</author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
      xml:
        attribute: true
    title:
      type: string
      xml:
        name: "xml-title"
    author:
      type: string
```

</TabItem>
</Tabs>

## 6. Namespace and prefix (inline form)

### 6.1 — Namespace on the model

`@Xml.ns` on a model adds a namespace prefix and declaration to the root element.

```tsp
@Xml.ns("http://example.com/schema", "smp")
model Book {
  id: integer;
  title: string;
  author: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema">
  <id>0</id>
  <title>string</title>
  <author>string</author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
    author:
      type: string
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
```

</TabItem>
</Tabs>

### 6.2 — Namespace on individual properties

Properties can declare their own namespace with `@Xml.ns`, which may differ from the model's namespace.

```tsp
@Xml.ns("http://example.com/schema", "smp")
model Book {
  id: integer;

  @Xml.ns("http://example.com/schema", "smp")
  title: string;

  @Xml.ns("http://example.com/ns2", "ns2")
  author: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema" xmlns:ns2="http://example.com/ns2">
  <id>0</id>
  <smp:title>string</smp:title>
  <ns2:author>string</ns2:author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
      xml:
        prefix: "smp"
        namespace: "http://example.com/schema"
    author:
      type: string
      xml:
        prefix: "ns2"
        namespace: "http://example.com/ns2"
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
```

</TabItem>
</Tabs>

## 7. Namespace and prefix (normalized form)

### 7.1 — Using an `@Xml.nsDeclarations` enum

Declare namespaces in an enum to avoid repeating URIs. Reference enum members with `@Xml.ns`.

```tsp
@Xml.nsDeclarations
enum Namespaces {
  smp: "http://example.com/schema",
}

@Xml.ns(Namespaces.smp)
model Book {
  id: integer;
  title: string;
  author: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema">
  <id>0</id>
  <title>string</title>
  <author>string</author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
    author:
      type: string
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
Namespaces:
  type: string
  enum:
    - http://example.com/schema
```

</TabItem>
</Tabs>

### 7.2 — Multiple namespaces via enum

Multiple namespace declarations can be defined in a single enum and referenced on individual properties.

```tsp
@Xml.nsDeclarations
enum Namespaces {
  smp: "http://example.com/schema",
  ns2: "http://example.com/ns2",
}

@Xml.ns(Namespaces.smp)
model Book {
  id: integer;

  @Xml.ns(Namespaces.smp)
  title: string;

  @Xml.ns(Namespaces.ns2)
  author: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema" xmlns:ns2="http://example.com/ns2">
  <id>0</id>
  <smp:title>string</smp:title>
  <ns2:author>string</ns2:author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml

  type: object
  properties:
    id:
      type: integer
    title:
      type: string
      xml:
        prefix: "smp"
        namespace: "http://example.com/schema"
    author:
      type: string
      xml:
        prefix: "ns2"
        namespace: "http://example.com/ns2"
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"

  type: string
  enum:
    - http://example.com/schema
    - http://example.com/ns2
```

</TabItem>
</Tabs>

## 8. Property setting the text of the node

### 8.1 — Text content with attributes

`@Xml.unwrapped` on a scalar property makes it the text content of the parent element. Combined with `@Xml.attribute`, you can model elements that have both attributes and inline text.

```tsp
model BlobName {
  @Xml.attribute language: string;
  @Xml.unwrapped content: string;
}
```

<Tabs>
<TabItem label="XML">

```xml
<BlobName language="abc">
  ...content...
</BlobName>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
BlobName:
  type: object
  properties:
    language:
      type: string
      xml:
        attribute: true
    content:
      type: string
```

</TabItem>
</Tabs>
